# Go 后端待修复项

## Token 统计能力

- 项目级 token 计算需要落到后端能力中。
  - 目标：为项目库页面提供项目级 CLI token 聚合统计。
  - 当前现状：`Project` 读模型与相关 API 还没有项目级 token 字段，前端只能做占位展示。
  - 预期输出：项目详情与项目列表接口返回可直接消费的项目级 token 使用量。

- 任务级 token 计算需要落到后端能力中。
  - 目标：为任务列表、任务详情、工作台等页面提供任务级 CLI token 聚合统计。
  - 当前现状：前端设计稿已经预留 token 展示位，但后端尚未形成统一的任务级 token 统计输出。
  - 预期输出：任务相关查询接口返回任务级 token 使用量，并保持字段命名稳定。

## 说明

- 这两项都属于后端真实能力缺口，不能继续由前端编造或硬编码。
- 实现时应优先在现有 Go 后端域模型与查询接口中补齐，而不是只做前端展示兼容。
